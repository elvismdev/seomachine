# Launch Checklists

## Pre-Launch Checklist (4-8 Weeks Before)

### Audience Building
- [ ] Email list / waitlist set up with landing page
- [ ] Early access signup with referral incentive
- [ ] Social presence established on 1-2 key platforms
- [ ] Community seeded (Slack/Discord or existing communities engaged)
- [ ] Beta testers recruited and onboarded

### Content Preparation
- [ ] Launch blog post drafted
- [ ] Email announcement sequence written (3-5 emails)
- [ ] Social media posts pre-written for launch day + week
- [ ] Product Hunt listing drafted (if applicable)
- [ ] Press/media outreach list compiled
- [ ] Demo video or walkthrough created

### Product Readiness
- [ ] Core features stable and tested
- [ ] Onboarding flow complete
- [ ] Pricing page live
- [ ] Help docs / knowledge base populated
- [ ] Feedback mechanism in-app
- [ ] Analytics tracking implemented

### Technical
- [ ] Performance tested under load
- [ ] Email deliverability verified
- [ ] Payment processing tested
- [ ] Error monitoring set up
- [ ] Backup and recovery plan

---

## Launch Day Checklist

### Morning (Before Announcement)
- [ ] Final smoke test of signup flow
- [ ] Payment processing verified
- [ ] Team aligned on support coverage
- [ ] Social media accounts ready
- [ ] Email sequence triggered or scheduled

### Launch Sequence
- [ ] Blog post published
- [ ] Email blast sent to waitlist
- [ ] Social media posts published
- [ ] Product Hunt submission (if applicable — launch at midnight PT)
- [ ] Hacker News / Reddit posts (if relevant)
- [ ] Personal outreach to key contacts

### Monitoring
- [ ] Analytics dashboard open (real-time)
- [ ] Error monitoring active
- [ ] Social media mentions tracked
- [ ] Support queue staffed
- [ ] Server performance monitored

### End of Day
- [ ] Respond to all Product Hunt comments
- [ ] Reply to all social mentions
- [ ] Address any critical bugs
- [ ] Thank early supporters
- [ ] Document key metrics

---

## Post-Launch Checklist (Week 1-4)

### Week 1
- [ ] Daily check-in on signup/activation metrics
- [ ] Respond to all user feedback
- [ ] Fix critical bugs within 24 hours
- [ ] Publish 2-3 follow-up social posts
- [ ] Send "getting started" email to new signups
- [ ] Compile user feedback themes

### Week 2-4
- [ ] Publish case study or early results post
- [ ] Iterate on top user feedback items
- [ ] Optimize conversion rate based on data
- [ ] Set up ongoing content calendar
- [ ] Establish regular feedback loop with users
- [ ] Plan next feature launch

## Product Hunt Launch Checklist

### 2 Weeks Before
- [ ] Profile optimized with real headshot and bio
- [ ] 5+ genuine community interactions (upvotes, comments)
- [ ] Hunter identified and confirmed (if using one)
- [ ] Tagline written (60 chars max, benefit-focused)
- [ ] Gallery images created (1200x900px, 5-8 images)
- [ ] First comment drafted (the maker's story)

### 1 Day Before
- [ ] All assets uploaded to Product Hunt draft
- [ ] Launch scheduled for midnight PT
- [ ] Email announcement ready to send at 9 AM PT
- [ ] Social posts scheduled
- [ ] Team briefed on commenting strategy

### Launch Day
- [ ] First comment posted immediately after launch
- [ ] Email sent to supporters asking for upvotes + comments
- [ ] Respond to every comment within 1 hour
- [ ] Share on social with direct link
- [ ] Monitor and engage throughout the day
- [ ] Post progress updates ("We hit #5!")

### Don't
- [ ] Don't ask for upvotes in PH comments (against rules)
- [ ] Don't use upvote pods or purchased votes
- [ ] Don't launch on Tuesday-Thursday without a reason (most competitive)
- [ ] Don't disappear after posting — engagement matters

## Feature Launch Checklist (Existing Product)

### Pre-Announcement
- [ ] Feature documented in changelog
- [ ] Help docs updated
- [ ] In-app announcement or banner prepared
- [ ] Email draft for relevant segment
- [ ] Social post(s) prepared

### Announcement
- [ ] Changelog entry published
- [ ] In-app notification triggered
- [ ] Email sent to relevant users
- [ ] Social media post published
- [ ] Blog post (if major feature)

### Follow-Up
- [ ] Monitor feature adoption rate
- [ ] Collect user feedback
- [ ] Address bugs within 24 hours
- [ ] Share usage metrics internally
